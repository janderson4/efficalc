Integrating and Extending efficalc#

Efficalc is designed to be a great standalone resource for building the perfect calculations. But we also appreciate the huge ecosystem of other helpful Python libraries that could work in conjunction with efficalc.

Not to mention the variety of modern workflows that engineers are coming up with everyday.

Here are some ideas and concepts that may help you integrate efficalc with your design workflows or extend the power of efficalc with your favorite libraries.

Parameters and Return Values in Calc Functions#

Efficalc is designed to simplify the creation of calculations by eliminating the necessity for parameters or return values. This approach is intended to make building calculations as easy as possible.

However, there might be scenarios where incorporating parameters as input values or producing output from your calculations is necessary to enable other workflows.

In this case, it’s best to provide a default value for each argument.

For example, if we want to calculate the hypotenuse of a right triangle, our calculation could look like this:

1def pythagorean_with_params(default_a=3, default_b=4):
2    a = Input("a", default_a)
3    b = Input("b", default_b)
4    c = Calculation("c", sqrt(a**2 + b**2), description="Length of the hypotenuse")
5    return c.result()

When running the ReportBuilder, this is equivalent to:

1def pythagorean_without_params():
2    a = Input("a", 3)
3    b = Input("b", 4)
4    c = Calculation("c", sqrt(a**2 + b**2), description="Length of the hypotenuse")

But, the calculation function with input parameters and a return value can also be used just like any other python function:

>>> print(pythagorean_with_params())
    5
>>> print(pythagorean_with_params(5, 6))
    7.810249675906654

With the ReportBuilder, we can still update our inputs just the same as shown in Change Input Values.

Hundreds of Calculations with One Function#

Here we’ll see a simple example of how one calculation function can be used to design hundreds of beams, columns, or whatever else you’re calculating. This really highlights the power of Python over other solutions like Excel.

In this example we will have one calculation and a collection of many inputs. The result will be a design for every input in the collection.

As a bonus, fixing an error in your calculation function will also fix the report and results for all designs that are based on that function.

For this example, we’ll use a calculation function with input parameters (see above).

Here’s a simplified example that finds the most efficient beam size for a set of constraints:

 1# Define a calc function for both optimization and the final report
 2def beam_strength(default_size="W10X12", default_span=12, default_Fy=50):
 3    Title("Beam Design")
 4
 5    # define inputs
 6    section_size = Input("size", default_size)
 7    span = Input("span", default_span, "ft")
 8    Fy = Input("F_y", default_Fy, "ksi")
 9
10    # get the section properties for the section size
11    size_name = section_size.get_value()
12    section = get_aisc_wide_flange(size_name)
13    Zx = Calculation("Z_x", section.Zx, "in^3")
14
15    # calculate the strength (more complex in real life)
16    strength = Calculation("M_p", Fy * Zx, "k-in")
17
18    # return information about this design for the optimization
19    return {
20        "size": size_name,
21        "weight": section.W,
22        "moment_strength": strength.result(),
23    }
24
25
26# now define an optimization function that uses beam_strength to find the optimal section size
27def find_lightest_beam_for_demand(size_options, moment_demand):
28    lightest_beam = {"size": "none", "weight": 999999, "moment_strength": 0}
29
30    # loop through all available sizes
31    for size in size_options:
32
33        # run the design calculation to get the strength of the section size
34        strength_info = beam_strength(size)
35        size_is_strong_enough = strength_info["moment_strength"] > moment_demand
36        size_is_lighter_than_best = strength_info["weight"] < lightest_beam["weight"]
37
38        # save the design output for this size if it's better than our previously saved section
39        if size_is_strong_enough and size_is_lighter_than_best:
40            lightest_beam = strength_info
41
42    # return the most efficient option we found
43    return lightest_beam["size"]
44
45
46# only some beam sizes may be available for a certain project, we want to find the most efficient one
47available_beam_sections = [
48    "W12X30",
49    "W12X26",
50    "W12X19",
51    "W12X14",
52    "W10X49",
53    "W10X33",
54    "W10X19",
55    "W10X12",
56    "W8X40",
57    "W8X21",
58    "W8X15",
59]
60
61moment_demand_on_beam = 1000  # k-in
62lightest_beam_size = find_lightest_beam_for_demand(
63    available_beam_sections, moment_demand_on_beam
64)
65
66# view calculation report for the lightest beam
67builder = ReportBuilder(beam_strength, {"size": lightest_beam_size})
68builder.view_report()

That was a lot to digest, but in summary this code:

  1. Defined a beam strength calculation function beam_strength

  2. Defined an optimization function (find_lightest_beam_for_demand) that uses the beam_strength calculation to find the lightest beam that is strong enough for the given demand

  3. Found the lightest available beam shape (by calling find_lightest_beam_for_demand)

  4. Generated the calculation report for our efficiently designed beam

Helper Functions#

Sometimes you may want to write helper functions that can be shared between multiple calculation functions. Or extracting a helper function might just help your calculation function be more clear and readable.

Helper functions can be utilized within calculations in 2 different ways:

Invisible Helpers#

These functions do not contain any efficalc calculation objects. Using this type of helper function will not show anything extra in a calculation report.

In this example:

1def invisible_square_sum(a: float, b: float):
2    return (a + b)**2
3
4def calculation():
5    a = Input("a", 3)
6    b = Input("b", 4)
7    Calculation("c", invisible_square_sum(a.value, b.value), description="An invisible result")

The calculation of variable “c” in this calculation report will simply show:

An invisible result
c = 49

Calculation Helpers#

These functions do contain efficalc calculation objects, so when you use this helper in a calculation function, the efficalc calculation objects within the function will be integrated into the calculation report.

In this example:

1def calculate_square_sum(a: Input | Calculation, b: Input | Calculation):
2    s = Calculation("sum", a + b)
3    return s**2
4
5
6def calculation():
7    a = Input("a", 3)
8    b = Input("b", 4)
9    Calculation("c", calculate_square_sum(a, b), description="A calculated result")

The calculation of variable “c” in this calculation report will now show:

sum = a + b = 3 + 4
    ∴ sum = 7

A calculated result
c = (sum)² = (7)²
  ∴ c = 49
